T-SEC-901 – Virology : Conception d’un outil C2 furtif (s0P0wn3d)

Date : Janvier 2026 – Aujourd'hui

Organisation : EPITECH – Projet T-SEC-901

Rôle : Offensive Security / Red Team (cadre pédagogique)

Description :
Projet universitaire de cybersécurité offensive consistant à concevoir et développer, from scratch, un prototype d’outil de Command & Control (C2) furtif nommé s0P0wn3d, afin de simuler des intrusions avancées et tester la capacité de détection d’un SOC dans un environnement contrôlé (VM Windows sur Azure).

Objectifs principaux :

  • Concevoir un agent C2 le plus furtif possible (évasion AV/EDR, réduction de la surface de logs).
  • Assurer la résilience de l’outil lors d’intrusions longues (communication fiable, persistance contrôlée).
  • Fournir un panel de commandes offensives pour évaluer les mécanismes de défense d’une organisation.

Fonctionnalités clés implémentées (en environnement de test uniquement) :

  • shell : ouverture d’un pseudo-shell basé sur des commandes natives.
  • keylog : activation / arrêt / extraction des frappes clavier à des fins d’étude.
  • rdp : activation / désactivation de Remote Desktop sur la machine cible.
  • crack & pth : lancement de tâches de craquage de mots de passe et attaques Pass-The-Hash pour comprendre les risques liés à une mauvaise gestion des secrets.
  • loot : recherche et exfiltration contrôlée de fichiers sensibles (KeePass, clés SSH, etc.) dans le lab.
  • phish : automatisation de scénarios de phishing à partir de la machine compromise (simulation).
  • propagate & privesc : étude de propagation latérale et d’élévation de privilèges locales.
  • syscall / whateveryouwant : exécution de syscalls depuis l’espace utilisateur et commande bonus pour expérimenter des techniques avancées.

Technologies & concepts : Windows (Azure), C2, réseau, cryptographie, appels système, détection & évasion de défenses, OpSec red team.

Livrables : Prototype fonctionnel testé sur une VM Azure, documentation technique et rapport de bonnes pratiques (perspective blue team : durcissement, détection, logging).

Cadre légal : Projet strictement pédagogique, réalisé dans un environnement isolé, avec autorisation explicite. Aucune utilisation en dehors d’un contexte d’audit formel et encadré.

GlycoPilot – Plateforme intelligente de suivi du diabète

Date : 2024 – 2025

Organisation : EPITECH

Rôle : Chargée de la sécurité de l’application & Développeuse backend

Description :
Application web & mobile de monitoring du diabète en temps réel : centralisation des données (glycémie, repas, activité), dashboards interactifs, alertes en temps réel et assistance intelligente pour améliorer le suivi patient et la prise de décision.

Contributions clés (sécurité) :

  • Conception et mise en place de l’authentification sécurisée (mots de passe hachés, gestion de session/JWT, protection CSRF/XSS).
  • Contrôle d’accès par rôles (patient, médecin, admin) et cloisonnement des données sensibles (PII).
  • Chiffrement des données en transit (TLS) et au repos, logs de sécurité et audit trail.
  • Application de bonnes pratiques OWASP (validation des entrées, rate limiting, en-têtes de sécurité, durcissement du déploiement).

Contributions clés (backend) :

  • Conception des schémas et APIs (collecte glycémies, repas, activité, alertes, statistiques).
  • Endpoints pour dashboards temps réel et génération de rapports.
  • Intégrations capteurs/exports et tâches asynchrones pour traitements périodiques.

Fonctionnalités majeures : Dashboards interactifs, alertes temps réel, suivi nutrition/activité, accessibilité renforcée, analyses multi-données et perspectives d’IA prédictive.

Technologies : Backend REST, base SQL, Auth JWT/Session, Docker, CI/CD (GitHub Actions), monitoring & logs.

Livrables : API documentée, guides de sécurité, plan de tests (fonctionnels & sécu), tableaux de bord.

TerraCloud – Continuous Configuration Automation (T-CLO-901)

Date : Sept. 2025 – Nov. 2025

Organisation : EPITECH

Rôle : Étudiante / Cloud & DevOps Engineer

Description :
Projet universitaire consistant à concevoir, déployer et comparer deux modèles d’infrastructure Cloud (IaaS et PaaS) sur Azure, avec un focus sur l’automatisation, la scalabilité et la maîtrise des coûts.

Missions principales :

  • Déploiement d’une application sur Azure avec deux approches : IaaS (machines virtuelles) et PaaS (Azure Web App).
  • Utilisation d’Infrastructure as Code avec Terraform pour créer et gérer les ressources.
  • Automatisation des déploiements avec Ansible et intégration de Docker.

Technologies : Azure, Terraform, Ansible, Docker, GitHub Actions.

Livrables : Guide de déploiement (IaaS & PaaS), documentation d’automatisation, rapport de comparaison coûts/performances.
