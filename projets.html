TEAM FORTRESS – DFIR : Network Forensics, Memory Forensics & Malware Reverse Engineering

Date : Mars 2026 – Juin 2026 · Organisation : EPITECH – Projet DFIR (Team Fortress) · Rôle : Analyste DFIR / Incident Response (cadre pédagogique)

Description : Projet universitaire de Digital Forensics & Incident Response (DFIR) consistant à investiguer 3 scénarios réalistes : analyse de trafic réseau (PCAP), analyse mémoire (dump RAM) et reverse engineering d'un binaire malveillant, afin d'identifier les IoC, reconstruire la timeline de l'attaque et produire un rapport DFIR professionnel.

Objectifs principaux :

  • Identifier l'activité malveillante, les canaux C2 et les tentatives d'exfiltration sur des captures réseau.
  • Détecter processus cachés, injection de code et mécanismes de persistance via l'analyse mémoire.
  • Comprendre les capacités d'un malware (comportements, IoC, logique interne) via reverse engineering.
  • Produire un rapport DFIR complet : méthodologie, preuves, IoC, timeline et recommandations.

Outils & méthodes : Wireshark, tcpdump, NetworkMiner, Volatility/Rekall, Ghidra/Cutter, strings, hash, timeline DFIR.

Livrables : Rapport DFIR (PDF) professionnel avec table des matières, IoC, timeline et recommandations de remédiation.

T-SEC-901 – Virology : Conception d'un outil C2 furtif (s0P0wn3d)

Date : Janvier 2026 – Aujourd'hui · Organisation : EPITECH – Projet T-SEC-901 · Rôle : Offensive Security / Red Team (cadre pédagogique)

Description : Projet universitaire de cybersécurité offensive consistant à concevoir et développer, from scratch, un prototype d'outil de Command & Control (C2) furtif nommé s0P0wn3d, afin de simuler des intrusions avancées et tester la capacité de détection d'un SOC dans un environnement contrôlé (VM Windows sur Azure).

Fonctionnalités clés implémentées (en environnement de test uniquement) :

  • shell : ouverture d'un pseudo-shell basé sur des commandes natives.
  • keylog : activation / arrêt / extraction des frappes clavier à des fins d'étude.
  • rdp : activation / désactivation de Remote Desktop sur la machine cible.
  • crack & pth : tâches de craquage de mots de passe et attaques Pass-The-Hash pour comprendre les risques liés à une mauvaise gestion des secrets.
  • loot : recherche et exfiltration contrôlée de fichiers sensibles (KeePass, clés SSH, etc.) dans le lab.
  • phish : automatisation de scénarios de phishing à partir de la machine compromise (simulation).
  • propagate & privesc : étude de propagation latérale et d'élévation de privilèges locales.
  • syscall / whateveryouwant : exécution de syscalls depuis l'espace utilisateur et commande bonus pour expérimenter des techniques avancées.

Technologies & concepts : Windows (Azure), C2, réseau, cryptographie, appels système, détection & évasion de défenses, OpSec red team.

Cadre légal : Projet strictement pédagogique, réalisé dans un environnement isolé, avec autorisation explicite.
